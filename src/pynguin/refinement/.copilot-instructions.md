# Copilot Instructions — Refinement Module (`pynguin/src/pynguin/refinement/`)

## Overview

This is the **TEST-LLM-REFINE** pipeline — the core thesis contribution. It is a post-processing extension to Pynguin that transforms raw machine-generated tests into human-readable, semantically enriched PyTest artifacts using LLM-based refinement.

**Entry point:** `refiner.py::refine_generated_tests()`, called from `generator.py` (~L654) when `--llm-refinement.enabled` is set.

---

## File-by-File Reference

### `refiner.py` — Public API (292 lines)

**Function:** `refine_generated_tests(test_file_path, module_name, ...)`

- Reads the generated test file, parses into `import_nodes` + `FunctionDef` nodes via `ast.parse()`
- Iterates each test function (up to `max_tests`, default 30)
- For each: extracts `original_code = import_block + ast.unparse(func)`, calls `TestRefiner.process_test_end_to_end()`
- Computes readability metrics before/after via `compute_all()`
- On failure, keeps original function unchanged
- Reassembles all functions into a new AST module, writes `{stem}_refined.py`
- Collects LLM usage via `refiner.llm_client.get_usage()` and wall time via `time.perf_counter()`
- Returns stats dict with keys: `tests_processed`, `tests_refined`, `repair_iterations`, `failed_tests`, `readability_original`, `readability_refined`, `readability_delta`, `llm_calls`, `llm_input_tokens`, `llm_output_tokens`, `wall_time_seconds`

**Key patterns:**
- `import_block` is reconstructed from all `Import`/`ImportFrom` nodes and prepended to each function
- Readability scores are averaged across all processed/refined tests
- The entire function is wrapped in try/except — never crashes the caller

### `pipeline.py` — Orchestrator (767 lines)

**Top-level function:** `_remove_failing_inferred_assertion(current_code, original_code, error_msg)` → `(modified_code, removed_assertion) | (None, None)`
- Uses `AssertionTracker` to diff original vs current assertions
- Walks AST to find and remove the first inferred assertion
- Returns `None, None` if no inferred assertion found

**Class:** `TestRefiner`

**`__init__(self, api_key, module_under_test, project_root, llm_model, subject_properties=None)`**
- Creates `LLMClient` (OpenAI)
- Creates `SUTInspector`
- Stores `module_under_test`, `project_root`, and `subject_properties`

**`structural_analysis(self, test_code) → dict`** — Stage 1
- Uses `FocalMethodAnalyzer(test_code, test_code).analyze()` → `FocalMethodInfo`
- If focal found: uses `SUTInspector.inspect_method()` to get docstring/signature, formats via `format_context_string()`
- Splits AST nodes into arrange/act/assert based on `focal_line_number`
- Falls back to `_fallback_structural_analysis()` on any error
- Returns dict: `test_name`, `focal_method_name`, `focal_line_number`, `sut_context`, `arrange`, `act`, `assert`, `full_code`

**`_fallback_structural_analysis(self, test_code) → dict`** — Legacy heuristic
- Last non-assertion statement = Act; everything before = Arrange; everything after = Assert
- Sets `focal_method_name = "unknown"`, `focal_line_number = 0`

**`refine_readability(self, analysis) → str`** — Stage 2
- Builds a detailed prompt with SUT context, current test, and requirements for:
  - Descriptive test function name (`test_<behavior_being_tested>`)
  - Semantic variable naming (from docstring context)
  - AAA structure with `# Arrange`, `# Act`, `# Assert` markers
  - Docstring addition
- Explicitly instructs: preserve behavior, don't add/remove assertions, keep imports
- Calls `self.llm_client.generate_code(prompt)`

**`generate_semantic_assertions(self, test_code, focal_method, sut_context) → str`** — Stage 2C
- Skips if `sut_context == "Documentation unavailable."`
- Builds a detailed prompt that:
  - Analyzes method contract from documentation
  - Distinguishes **positive tests** (add `isinstance`, `len`, etc.) from **negative tests** (wrap in `pytest.raises()`)
  - Provides examples for STRING, INTEGER, BOOLEAN, DICT/LIST return types
  - Instructs to preserve AAA markers, variable names, and imports
- Returns original code if generation fails or response lacks imports

**`repair_test_code(self, broken_code, error_message) → str`** — Stage 3 repair
- Prompt: "Fix the test code to resolve the error" with broken code + traceback
- Handles: SyntaxError, ImportError, NameError, TypeError, AssertionError
- Returns original code if repair fails

**`process_test_end_to_end(self, original_code, max_retries=3) → dict`** — Full pipeline
1. **Level 1 baseline** — `run_test(original_code)` to verify original passes (warning if not)
2. **Stage 1** — `structural_analysis(original_code)` → focal method + SUT context
3. **Stage 2** — `refine_readability(analysis)` → semantic naming + structure
4. **Stage 2C** — `generate_semantic_assertions(readable_code, focal_method, sut_context)` → assertion strengthening
5. **Stage 3, Phase 1** — Repair loop (`max_retries + 1` iterations):
   - `run_test(current_code)` → if pass, break
   - Classify error: SyntaxError, ImportError, NameError, AssertionError, TypeError
   - **Assertion-failure policy**: if `AssertionError` on inferred assertion → `_remove_failing_inferred_assertion()` auto-discard (doesn't count as repair iteration, uses `continue`)
   - Otherwise → `repair_test_code()` via LLM
6. **Stage 3, Phase 2** — `filter_vacuous_assertions()` with mutation testing
   - If test fails after filtering → reverts to pre-filter code (`assertion_code`)
7. **Level 2** — `check_coverage_preservation()` with `tolerance=0.0` and `subject_properties=self.subject_properties`
   - Uses Pynguin's `ExecutionTracer` + `compute_branch_coverage()` when `subject_properties` available
   - Falls back to `sys.settrace` line coverage in standalone mode
   - If coverage drops → returns `success: False`
8. **Post-repair** — `insert_aaa_markers_simple()`:
   - Lazy import: `from pynguin.refinement.aaa_inserter import insert_aaa_markers_simple`
   - Re-analyzes focal method position on final code before inserting markers

Returns dict: `success`, `final_code`, `iterations`, `mutation_stats`, `coverage_details`, or `error`

### `ast_analyzer.py` — Focal Method Detection (271 lines)

**Dataclass:** `FocalMethodInfo`
- `focal_method_name`: e.g., `"logger.bind"` or `"bind"`
- `focal_line_number`: line number in test source
- `module_alias`: e.g., `"logger"`, `"module_0"`
- `resolved_module_name`: e.g., `"structlog"` (via import map)
- `full_call_signature`: e.g., `"logger.bind(key='value')"`

**Class:** `ImportMapBuilder(ast.NodeVisitor)`
- Builds `Dict[str, str]` mapping alias → module name
- Handles `import foo as bar`, `from foo import Bar as Baz`, `import foo`

**Class:** `CallAndAssertCollector(ast.NodeVisitor)`
- Collects `calls: List[Tuple[ast.Call, int]]` (call node + line number)
- Records `first_assert_line: Optional[int]`
- Visits nested structures (try/except, with, if/else, loops)

**Class:** `FocalMethodAnalyzer`
- `__init__(test_function_source, full_file_source=None)` — full file source for import map
- `_build_import_map()` → uses `ImportMapBuilder`
- `_extract_call_info(call_node)` → `(method_name, module_alias)` — handles `obj.method()`, `func()`, chained calls
- `_resolve_module_name(alias)` → actual module name from import map
- `_find_focal_method()` → **core algorithm**: last `Call` before first `Assert`; falls back to last `Call` if no assertion
- `analyze()` → `Optional[FocalMethodInfo]` — builds import map, then finds focal method

### `llm_client.py` — OpenAI LLM Wrapper (278 lines)

**Class:** `LLMClient`
- `__init__(model_name, api_key)`

- Loads `.env` via `python-dotenv` (searches up to 5 parent dirs)

**`generate_code(self, prompt) → str`**

  - **Callers MUST check for `"# LLM error"` prefix**

**`_extract_code(self, text) → str`**
- Extracts content from ` ```python ... ``` ` blocks first, then ` ``` ... ``` `, then returns raw text

### `mutation_analyzer.py` — Per-Assertion Mutation Filtering (418 lines)

**Class:** `AssertionTracker`
- `__init__(original_test, refined_test)` — parses both, identifies `inferred_assertions` (in refined but not in original)
- `_extract_assertions(test_code)` → list of `ast.unparse(node.test)` strings
- `_identify_new_assertions()` → set difference (refined - original)

**Helper functions:**
- `_run_test_against_mutant(test_code, mutant_module, module_name)` → `True` if mutant killed (test raises exception)
  - Uses `exec()` with `test_globals` namespace containing mutant module
- `_remove_assertion_by_index(tree, target_idx)` → deep copy with target Assert replaced by `pass`
- `_killed_set(test_code, mutants, module_name)` → `set[int]` of killed mutant indices

**`filter_vacuous_assertions(original_test, refined_test, focal_method, module_under_test, module_path, max_mutants=10, other_tests_in_suite=None) → (str, dict)`**
1. **Step 1:** Uses `AssertionTracker` to identify inferred assertions. Returns early if none.
2. **Step 2:** Imports Pynguin's mutation infrastructure:
   - `MutationController`, `FirstOrderMutator`, `ParentNodeTransformer`
   - Operators: `ArithmeticOperatorReplacement`, `RelationalOperatorReplacement`, `LogicalOperatorReplacement`, `ConstantReplacement`
   - Generates up to `max_mutants` (default 10) mutants from SUT AST
   - NOTE: Creates a fresh `MutationController` — deterministic, same operators produce identical mutant set
3. **Step 3:** Dual-level per-assertion filtering:
   - Computes `baseline_killed` with all assertions
   - **If `other_tests_in_suite` provided:** Computes `suite_baseline_killed` (mutants killed by all OTHER tests)
   - For each inferred assertion index:
     - Removes it, computes `without_killed`
     - **Per-test contribution:** `additional_kills = baseline_killed - without_killed`
     - **Suite-level contribution:** `unique_to_assertion = additional_kills - suite_baseline_killed` (mutants killed ONLY by this assertion, not by any other test)
     - **Filtering criterion:** If `len(additional_kills) == 0` → assertion is vacuous (per-test), mark for removal
     - **Reporting:** Both per-test and suite-level contributions recorded in stats
4. **Step 4:** Builds filtered test using `_BulkRemover` (replaces removed asserts with `pass`)
- Returns `(filtered_test_code, stats_dict)` with stats: `inferred_assertions`, `mutants_generated`, `mutants_killed_total`, `assertions_kept`, `assertions_removed`, **`per_test_contributions`** (dict), **`suite_level_contributions`** (dict), **`suite_baseline_size`**, **`avg_per_test_contribution`**, **`avg_suite_level_contribution`**
- **Design rationale:** Per-test filtering is conservative (runs during refinement when suite may be incomplete). Suite-level metrics enable evaluation of cross-test redundancy without affecting filtering decisions.

### `validator.py` — In-Process Test Execution (35 lines)

**`run_test(test_code, module_under_test) → (bool, str)`**
- Creates scope with `{module.__name__: module}`
- Finds function name by scanning for `def ` lines
- `exec()` the code, then calls the function
- Returns `(True, "Test passed.")` or `(False, error_with_traceback)`
- Catches `AssertionError` separately, then `BaseException` (includes `pytest.fail`)
- **WARNING:** Side effects leak into the calling process (no subprocess isolation)

### `coverage_checker.py` — Coverage Preservation (271 lines)

**Dataclass:** `CoverageResult`
- `coverage_value: float` (0.0–1.0), `metric: str` ("branch" or "line"), `error: str | None`

**`_measure_coverage_pynguin(test_code, module_under_test, subject_properties) → CoverageResult`**
- Primary path: uses Pynguin's `ExecutionTracer` with bytecode-instrumented module
- `tracer.init_trace()` → `tracer.temporarily_enable()` → `exec()` → `tracer.get_trace()` → `compute_branch_coverage()`
- Uses configured coverage metric (branch by default, line if configured)

**`_executable_lines(source) → set[int]`**
- Parses SUT source, walks AST, collects line numbers of executable statements
- Skips structural nodes (FunctionDef, ClassDef, Import, ImportFrom)

**`_measure_coverage_settrace(test_code, module_under_test) → CoverageResult`**
- Fallback for standalone mode (no `SubjectProperties` available)
- Uses `sys.settrace()` with a custom tracer that records line events matching the SUT file
- Thread-safe via `threading.Lock`

**`check_coverage_preservation(original_test, refined_test, module_under_test, tolerance=0.0, subject_properties=None) → (bool, dict)`**
- Dispatches to Pynguin-native or `sys.settrace` fallback based on `subject_properties` availability
- Measures coverage for both original and refined
- Passes if `refined_coverage >= original_coverage - tolerance`
- Skips (returns True) if original coverage can't be measured
- Returns details dict with `metric`, `original_coverage`, `refined_coverage`, `coverage_delta`, `status`

### `aaa_inserter.py` — AAA Marker Insertion (131 lines)

**`insert_aaa_markers_simple(test_code, focal_line_number, verbose=True) → str`**
- Strips existing `# Arrange`, `# Act`, `# Assert` markers
- Finds `def test_*` function, locates body start (past docstrings)
- Finds `first_assert_idx` by scanning for lines starting with `assert`
- Sets `focal_idx` as the last non-empty line before first assert
- Inserts `    # Arrange` before arrange section, `    # Act` before focal call, `    # Assert` before assertions
- Uses 4-space indentation for markers
- Returns original code on any error

### `readability_metrics.py` — Heuristic Scoring (97 lines)

**Constants:**
- `_VAR_GENERIC_PATTERN = re.compile(r"^(var|list|dict|tuple|set)_\d+$")`

**Scoring functions (all return 0.0–1.0):**
- `score_aaa(test_code)` — checks presence of `# arrange`, `# act`, `# assert` (case-insensitive), bonus for correct order
- `score_semantic_names(test_code)` — ratio of non-generic identifiers (ast.Name + ast.arg)
- `score_assertion_clarity(test_code)` — ratio of "specific" asserts (Compare, Call, Attribute, Subscript) vs total
- `score_conciseness(test_code)` — ideal window 10–40 lines (excluding blanks/comments), decay outside

**`compute_all(test_code) → dict`** — returns all 4 scores + `aggregate` (arithmetic mean)

**Dataclass:** `JudgePairResult`

**Class:** `LLMJudge`

## Architecture Diagram

```
refiner.py (public API)
  └─ pipeline.py::TestRefiner.process_test_end_to_end()
       ├── [Level 1] validator.py::run_test()           ← baseline check
       ├── [Stage 1] ast_analyzer.py::FocalMethodAnalyzer.analyze()
       │                + sut_inspector.py::SUTInspector.inspect_method()
       ├── [Stage 2] pipeline.py::refine_readability()  ← LLM call via llm_client.py
       ├── [Stage 2C] pipeline.py::generate_semantic_assertions()  ← LLM call
       ├── [Stage 3.1] Repair loop:
       │     ├── validator.py::run_test()
       │     ├── pipeline.py::_remove_failing_inferred_assertion()  ← auto-discard
       │     └── pipeline.py::repair_test_code()        ← LLM call
       ├── [Stage 3.2] mutation_analyzer.py::filter_vacuous_assertions()
       ├── [Level 2] coverage_checker.py::check_coverage_preservation()
       └── [Post] aaa_inserter.py::insert_aaa_markers_simple()
```

Readability is measured at entry/exit:
- `readability_metrics.py::compute_all()` — heuristic scores (computed inside the pipeline)
- `llm_judge.py::LLMJudge` — standalone post-pipeline evaluation tool (RQ1, run separately)

## Key Design Decisions

### Assertion-failure policy
When a test fails with `AssertionError` on an **inferred** assertion (one added by the LLM, not in the original), the pipeline **auto-discards** that assertion rather than sending it to the LLM for repair. This prevents the LLM from weakening expectations or altering SUT setup to force the assertion to pass. The discard does not count as a repair iteration.

### Mutation filtering reuses Pynguin infrastructure
`filter_vacuous_assertions()` imports Pynguin's `MutationController`, `FirstOrderMutator`, and four operators directly. It creates a fresh controller with the same operators and module AST, producing a deterministic identical mutant set — functionally equivalent to reusing cached mutants without cross-phase plumbing.

### Coverage check uses Pynguin's instrumentation
When `SubjectProperties` are available, coverage is measured using Pynguin's own `ExecutionTracer` and `compute_branch_coverage()` — the same bytecode-instrumented infrastructure used during test generation. The SUT module is already loaded with instrumented bytecode; the tracer is temporarily re-enabled to measure each test's coverage. Falls back to `sys.settrace` line coverage in standalone mode.

### AAA markers are always programmatic
AAA comments are **never** included in LLM prompts for output. They are stripped and re-inserted deterministically by `insert_aaa_markers_simple()` based on the post-repair focal method position.

### Error sentinels from LLM
`LLMClient.generate_code()` returns `"# LLM error: ..."` strings on failure. All callers should check for this prefix before using the result.

## Important Caveats

1. **Windows:** `sut_inspector.py`'s `time_limit()` is a no-op — `SIGALRM` unavailable
2. **Side effects:** `validator.py` uses `exec()` in-process — side effects leak
3. **No unit tests:** This module has no tests under `pynguin/tests/`. Ad-hoc tests exist at workspace root (`test_aaa_insertion.py`, `test_local_refinement.py`, etc.)
4. **Bare imports in pipeline.py:** `from pynguin.refinement.aaa_inserter import insert_aaa_markers_simple` is done as a lazy import inside `process_test_end_to_end()`
5. **`ast.unparse()` dependency:** Used extensively — requires Python 3.9+; some older formatting may be lost
6. **Token tracking:** Uses OpenAI usage data when provided by the API
7. **`max_mutants` default:** Only 10 mutants generated for filtering — keeps cost low but may miss some vacuous assertions
8. **Generic pattern:** `readability_metrics.py` only matches `var|list|dict|tuple|set` prefix — doesn't catch `str_0`, `int_0`, `bool_0`, `float_0`, `bytes_0`, `obj_0`, `module_0` (known limitation vs. proposal's broader pattern)

## Adding or Modifying Code

- Follow Pynguin conventions: `from __future__ import annotations`, Google docstrings, type hints, ruff-formatted
- All dataclass returns should have a `success` or `error` field for robustness
- New LLM prompts go in `pipeline.py` as methods on `TestRefiner`
- New analysis stages: add a method to `TestRefiner`, integrate into `process_test_end_to_end()`
- New metrics: add a scoring function to `readability_metrics.py`, include in `aggregate_readability()`
- Stats tracking: any new stats should be added to the return dict in `refiner.py` and tracked in `generator.py`'s `RuntimeVariable` mapping
